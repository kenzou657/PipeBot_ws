# 蓝牙控制系统文档

## 系统架构

```
手机蓝牙应用（Android/iOS）
    ↓ (蓝牙 RFCOMM/BLE)
bluetooth_receiver_node  ← 使用 Bless 库接收命令
    ↓ (bluetooth/command 话题)
command_handler_node     ← 解析命令 + 看门狗
    ↓ (cmd_vel 话题)
serial_node              ← 发送到 STM32
    ↓ (串口)
STM32 下位机

同时：
state_manager_node       ← 收集所有传感器状态
    ↓ (robot/state 话题)
bluetooth_sender_node    ← 构造反馈数据包
    ↓ (蓝牙 RFCOMM/BLE)
手机蓝牙应用
```

## 蓝牙库选择

本项目使用 **Bless** 库作为蓝牙通信的基础：

- **Bless**：现代、跨平台、支持 RFCOMM 和 BLE
- **优点**：安装简单、树莓派 4B 完美支持、异步 API
- **支持**：经典蓝牙（HC-05/HC-06）和 BLE 低功耗蓝牙

### 安装 Bless

```bash
# 树莓派上安装
pip3 install bless

# 验证安装
python3 -c "from bless import BlessServer; print('✓ Bless installed')"
```

## 命令协议

### 上行命令（手机 → 树莓派）

| 命令名称 | 命令码 | 参数 | 说明 |
|---------|--------|------|------|
| 切换自动模式 | 0x10 | 无 | 切换到自动模式，依赖测距模块 |
| 切换手动模式 | 0x11 | 无 | 切换到手动模式，遥控操作 |
| 前进 | 0x20 | [速度] | 手动模式：前进，速度 0-255 |
| 后退 | 0x21 | [速度] | 手动模式：后退，速度 0-255 |
| 制动 | 0x22 | 无 | 手动模式：停止运动 |
| 紧急停止 | 0x23 | 无 | 两种模式：立即停止 |
| 左转 | 0x24 | 无 | 手动模式：原地左转 |
| 右转 | 0x25 | 无 | 手动模式：原地右转 |

**命令格式**：
```
简单命令：[命令码]
带参数命令：[命令码][参数1][参数2]...
```

**示例**：
```python
# 切换到自动模式
send_command(b'\x10')

# 前进，速度 150
send_command(b'\x20\x96')

# 紧急停止
send_command(b'\x23')
```

### 下行反馈（树莓派 → 手机）

**数据包格式**：
```
[0xAA] [数据类型] [数据长度] [数据] [校验位] [0xBB]
```

| 数据类型 | 说明 | 数据格式 |
|---------|------|---------|
| 0x01 | 位置信息 | x(float), y(float), theta(float) |
| 0x02 | 速度信息 | linear(float), angular(float) |
| 0x03 | 测距信息 | left_dist(uint16), right_dist(uint16) |
| 0x04 | 模式状态 | mode(uint8), bt_status(uint8) |
| 0x05 | 警告信息 | warning_text(string) |

**示例解析**（Python）：
```python
def parse_feedback(data):
    if data[0] != 0xAA or data[-1] != 0xBB:
        return None
    
    data_type = data[1]
    data_len = data[2]
    payload = data[3:3+data_len]
    checksum = data[3+data_len]
    
    # 验证校验位
    if sum(data[:3+data_len]) & 0xFF != checksum:
        return None
    
    if data_type == 0x01:  # 位置信息
        import struct
        x, y, theta = struct.unpack('<fff', payload)
        return {'type': 'position', 'x': x, 'y': y, 'theta': theta}
    
    # ... 其他类型解析
```

## 运行模式

### 1. 自动模式
- **触发**：发送 `0x10` 命令
- **行为**：
  - 机器人自动前进
  - 根据测距模块数据避障
  - 底层 STM32 负责路径规划
- **控制**：
  - 只能发送 `0x23`（紧急停止）
  - 其他运动命令被忽略

### 2. 手动模式
- **触发**：发送 `0x11` 命令
- **行为**：
  - 完全由手机控制
  - 长按发送连续命令
  - 松开自动制动（看门狗）
- **控制**：
  - `0x20`：前进
  - `0x21`：后退
  - `0x24`：左转
  - `0x25`：右转
  - `0x22`：制动
  - `0x23`：紧急停止

## 安全机制

### 1. 看门狗定时器
- **超时时间**：500ms（可配置）
- **触发条件**：手动模式下 500ms 未收到命令
- **动作**：自动制动

### 2. 蓝牙断线保护
- **检测**：`state_manager_node` 监控连接状态
- **动作**：发出警告，建议切换到安全模式

### 3. 测距警告
- **阈值**：200mm（可配置）
- **触发**：左右测距传感器检测到障碍物
- **动作**：发送警告到手机

## 手机应用开发指南

### Android（Kotlin）- 经典蓝牙

```kotlin
import android.bluetooth.BluetoothAdapter
import android.bluetooth.BluetoothSocket
import java.util.*

class BluetoothController {
    private val bluetoothAdapter = BluetoothAdapter.getDefaultAdapter()
    private var socket: BluetoothSocket? = null
    private var inputStream = socket?.inputStream
    private var outputStream = socket?.outputStream
    
    // 树莓派蓝牙地址（需要替换为实际地址）
    private val PIPEBOT_ADDRESS = "B8:27:EB:XX:XX:XX"
    private val SERIAL_UUID = UUID.fromString("00001101-0000-1000-8000-00805F9B34FB")
    
    fun connect() {
        val device = bluetoothAdapter.getRemoteDevice(PIPEBOT_ADDRESS)
        socket = device.createRfcommSocketToServiceRecord(SERIAL_UUID)
        socket?.connect()
        inputStream = socket?.inputStream
        outputStream = socket?.outputStream
    }
    
    // 发送命令
    fun sendCommand(command: ByteArray) {
        outputStream?.write(command)
        outputStream?.flush()
    }
    
    // 接收反馈
    fun receiveFeedback(): ByteArray? {
        val buffer = ByteArray(256)
        return try {
            val len = inputStream?.read(buffer) ?: 0
            if (len > 0) buffer.copyOf(len) else null
        } catch (e: Exception) {
            null
        }
    }
    
    // 便捷方法
    fun setAutoMode() = sendCommand(byteArrayOf(0x10))
    fun setManualMode() = sendCommand(byteArrayOf(0x11))
    fun forward(speed: Int = 150) = sendCommand(byteArrayOf(0x20, speed.toByte()))
    fun backward(speed: Int = 150) = sendCommand(byteArrayOf(0x21, speed.toByte()))
    fun brake() = sendCommand(byteArrayOf(0x22))
    fun emergencyStop() = sendCommand(byteArrayOf(0x23))
    fun turnLeft() = sendCommand(byteArrayOf(0x24))
    fun turnRight() = sendCommand(byteArrayOf(0x25))
    
    fun disconnect() {
        socket?.close()
    }
}
```

### iOS（Swift）- BLE

```swift
import CoreBluetooth

class BluetoothController: NSObject, CBCentralManagerDelegate, CBPeripheralDelegate {
    var centralManager: CBCentralManager?
    var peripheral: CBPeripheral?
    var characteristic: CBCharacteristic?
    
    override init() {
        super.init()
        centralManager = CBCentralManager(delegate: self, queue: nil)
    }
    
    func startScanning() {
        centralManager?.scanForPeripherals(withServices: nil)
    }
    
    func sendCommand(_ command: Data) {
        guard let peripheral = peripheral, let characteristic = characteristic else { return }
        peripheral.writeValue(command, for: characteristic, type: .withResponse)
    }
    
    // MARK: - CBCentralManagerDelegate
    func centralManager(_ central: CBCentralManager, didDiscover peripheral: CBPeripheral, advertisementData: [String : Any], rssi RSSI: NSNumber) {
        if peripheral.name == "PipeBot Control" {
            self.peripheral = peripheral
            central.connect(peripheral)
        }
    }
}
```

## 故障排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| Bless 无法安装 | 缺少系统依赖 | `sudo apt install libbluetooth-dev` |
| 蓝牙无法连接 | 树莓派蓝牙未启动 | `sudo systemctl start bluetooth` |
| 命令无响应 | 节点未运行 | 检查所有节点是否启动 |
| 机器人自动制动 | 看门狗超时 | 增加命令发送频率（< 500ms） |
| 收不到反馈 | sender 节点未运行 | 启动 `bluetooth_sender_node` |
