# 蓝牙控制系统架构文档

## 系统概述

PipeBot 蓝牙控制系统使用 **python-bluezero** 库实现标准 BLE（蓝牙低功耗）服务，支持手机通过 nRF Connect 等 BLE 应用进行远程控制。

## 技术栈

| 组件 | 技术 | 版本 |
|------|------|------|
| **BLE 库** | python-bluezero | >= 0.6.0 |
| **ROS2** | ROS2 Humble | - |
| **Python** | Python 3.10+ | - |
| **操作系统** | Linux (树莓派) | - |

## 架构设计

### 系统架构图

```
┌─────────────────────────────────────────┐
│         手机端 (BLE 客户端)              │
│      iOS/Android nRF Connect            │
└────────────────┬────────────────────────┘
                 │ BLE 无线通信
                 ▼
┌─────────────────────────────────────────┐
│    bluetooth_receiver_node (主线程)      │
│  ┌───────────────────────────────────┐  │
│  │ 守护线程 (同步操作)                │  │
│  │ ┌─────────────────────────────┐   │  │
│  │ │ Peripheral (bluezero)       │   │  │
│  │ │ - GATT 服务                 │   │  │
│  │ │ - 特征值读写                │   │  │
│  │ │ - 广播管理                  │   │  │
│  │ └─────────────────────────────┘   │  │
│  └───────────────────────────────────┘  │
│                 │                        │
│  发布:          ▼                        │
│  bluetooth/    _on_write_request()       │
│  command       └─► bluetooth/command    │
│                                          │
│  订阅:                                   │
│  bluetooth/    feedback_callback()       │
│  feedback      └─► BLE 通知              │
└─────────────────────────────────────────┘
         │
         ├─► command_handler_node
         ├─► state_manager_node
         └─► bluetooth_sender_node
              └─► bluetooth/feedback
```

### 节点说明

#### 1. bluetooth_receiver_node（蓝牙接收节点）

**职责：**
- 创建 BLE 外设并定义 GATT 服务
- 监听手机连接/断开事件
- 接收手机发送的命令
- 发送反馈数据到手机

**发布话题：**
- `bluetooth/status` - 连接状态（waiting/connected/disconnected）
- `bluetooth/command` - 接收到的命令数据

**订阅话题：**
- `bluetooth/feedback` - 反馈数据

**参数：**
```yaml
bt_name: 'PipeBot Control'                    # 蓝牙设备名称
service_uuid: '12345678-1234-5678-1234-56789ABCDEF0'  # 服务 UUID
rx_char_uuid: '12345678-1234-5678-1234-56789ABCDEF1'  # 接收特征值 UUID
tx_char_uuid: '12345678-1234-5678-1234-56789ABCDEF2'  # 发送特征值 UUID
```

#### 2. bluetooth_sender_node（蓝牙发送节点）

**职责：**
- 订阅机器人状态
- 构造反馈数据包
- 发布到 `bluetooth/feedback` 话题

**订阅话题：**
- `robot/state` - 机器人完整状态
- `bluetooth/status` - 蓝牙连接状态

**发布话题：**
- `bluetooth/feedback` - 反馈数据

#### 3. command_handler_node（命令处理节点）

**职责：**
- 接收蓝牙命令
- 解析命令类型
- 发送速度控制命令

**订阅话题：**
- `bluetooth/command` - 蓝牙命令

**发布话题：**
- `cmd_vel` - 速度命令
- `robot/mode` - 模式状态

#### 4. state_manager_node（状态管理节点）

**职责：**
- 聚合所有传感器数据
- 生成完整的机器人状态
- 定期发布状态信息

**订阅话题：**
- `odom` - 里程计数据
- `imu/data_raw` - IMU 数据
- `sensor/distance` - 测距数据
- `motor/feedback` - 电机反馈
- 等等

**发布话题：**
- `robot/state` - 完整状态（JSON 格式）

## GATT 服务定义

### 服务结构

```
Service: 12345678-1234-5678-1234-56789ABCDEF0
├── Characteristic: 12345678-1234-5678-1234-56789ABCDEF1 (RX)
│   ├── Properties: write, write-without-response
│   ├── Max Size: 512 bytes
│   └── Purpose: 接收手机命令
│
└── Characteristic: 12345678-1234-5678-1234-56789ABCDEF2 (TX)
    ├── Properties: read, notify
    ├── Max Size: 512 bytes
    └── Purpose: 发送反馈数据
```

### 特征值说明

#### 接收特征值 (RX)

**UUID:** `12345678-1234-5678-1234-56789ABCDEF1`

**属性：** write, write-without-response

**用途：** 接收手机发送的命令

**命令格式：**
```
[命令类型] [参数...]

命令类型：
0x10 = 自动模式
0x11 = 手动模式
0x20 = 前进 (参数: 速度 0-255)
0x21 = 后退 (参数: 速度 0-255)
0x22 = 制动
0x23 = 停止
0x24 = 左转
0x25 = 右转
```

#### 发送特征值 (TX)

**UUID:** `12345678-1234-5678-1234-56789ABCDEF2`

**属性：** read, notify

**用途：** 发送反馈数据到手机

**反馈格式：**
```
[0xAA] [数据类型] [数据长度] [数据] [校验位] [0xBB]

数据类型：
0x01 = 位置信息 (x, y, theta)
0x02 = 速度信息 (linear, angular)
0x03 = 测距信息 (left, right)
0x04 = 模式状态 (mode, bt_status)
0x05 = 警告信息 (warning_text)
```

## 数据流向

### 命令流向

```
手机 (nRF Connect)
  │ 写入接收特征值
  ▼
bluetooth_receiver_node
  │ 发布 bluetooth/command
  ▼
command_handler_node
  │ 发布 cmd_vel, robot/mode
  ▼
其他节点处理
  │ 发布 odom, imu, sensor/distance
  ▼
state_manager_node
  │ 发布 robot/state
  ▼
bluetooth_sender_node
  │ 发布 bluetooth/feedback
  ▼
bluetooth_receiver_node
  │ 发送 BLE 通知
  ▼
手机 (nRF Connect)
```

## 线程模型

### 主线程
- 运行 ROS2 节点
- 处理订阅/发布
- 管理生命周期

### 守护线程
- 运行 BLE 外设
- 处理 BLE 事件
- 同步操作（无 asyncio）

### 线程通信
- 使用 ROS2 话题进行线程间通信
- 避免直接的线程间调用

## 配置参数

### bluetooth_receiver_node

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `bt_name` | `PipeBot Control` | 蓝牙设备名称 |
| `service_uuid` | `12345678-1234-5678-1234-56789ABCDEF0` | GATT 服务 UUID |
| `rx_char_uuid` | `12345678-1234-5678-1234-56789ABCDEF1` | 接收特征值 UUID |
| `tx_char_uuid` | `12345678-1234-5678-1234-56789ABCDEF2` | 发送特征值 UUID |

### state_manager_node

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `publish_rate` | `2.0` | 状态发布频率（Hz） |
| `distance_warning_threshold` | `200.0` | 测距警告阈值（mm） |

### command_handler_node

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `command_timeout` | `0.5` | 命令超时时间（秒） |
| `default_speed` | `100` | 默认速度（0-255） |
| `turn_speed` | `80` | 转向速度（0-255） |

### bluetooth_sender_node

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `feedback_rate` | `2.0` | 反馈发送频率（Hz） |

## 启动方式

### 使用启动文件

```bash
ros2 launch PipeBot_bringup bluetooth_control.launch.py
```

### 手动启动各节点

```bash
# 终端 1
ros2 run PipeBot_driver bluetooth_receiver_node

# 终端 2
ros2 run PipeBot_driver state_manager_node

# 终端 3
ros2 run PipeBot_driver command_handler_node

# 终端 4
ros2 run PipeBot_driver bluetooth_sender_node
```

## 手机端操作

### 使用 nRF Connect

1. 打开 nRF Connect 应用
2. 点击 "Scan" 扫描设备
3. 找到 "PipeBot Control" 设备
4. 点击连接
5. 查看 GATT 服务和特征值
6. 向接收特征值写入命令
7. 从发送特征值读取反馈

## 故障排查

### 常见问题

**问题 1：蓝牙设备不显示**
- 检查蓝牙硬件是否启用
- 检查节点是否正常运行
- 查看日志输出

**问题 2：连接后立即断开**
- 检查 UUID 是否正确
- 检查特征值属性是否正确
- 查看蓝牙日志

**问题 3：无法发送/接收数据**
- 检查特征值是否正确定义
- 检查数据格式是否正确
- 查看 ROS2 话题是否有数据

### 调试命令

```bash
# 查看蓝牙设备
hciconfig

# 查看 ROS2 话题
ros2 topic list

# 监听话题
ros2 topic echo bluetooth/command
ros2 topic echo bluetooth/feedback

# 查看节点信息
ros2 node list
ros2 node info bluetooth_receiver_node
```
