## 环境准备

### 1. 检查蓝牙硬件

在 Ubuntu 终端中执行：

```bash
# 检查蓝牙适配器
hciconfig

# 预期输出：
# hci0:   Type: Primary  Bus: USB
#         UP RUNNING
#         BD Address: XX:XX:XX:XX:XX:XX  ACL MTU: 310:10  SCO MTU: 64:8
```

如果蓝牙未启用，执行：

```bash
sudo hciconfig hci0 up
```

检查蓝牙权限：

```bash
# 查看当前用户是否在 bluetooth 组
groups $USER

# 如果不在 bluetooth 组，添加用户
sudo usermod -a -G bluetooth $USER

# 重新登录或运行
newgrp bluetooth
```

### 2. 构建项目

```bash
cd /home/pi/PipeBot_ws

# 构建整个工作空间
colcon build

# 或仅构建 PipeBot_driver 和 PipeBot_bringup
colcon build --packages-select PipeBot_driver PipeBot_bringup

# 源环境
source install/setup.bash
```

**预期输出：**
```
Starting >>> PipeBot_driver
Finished <<< PipeBot_driver [1.23s]
Starting >>> PipeBot_bringup
Finished <<< PipeBot_bringup [0.45s]
```

### 3. 准备手机

- 在手机上安装 **nRF Connect** 应用（iOS 或 Android）
- 确保手机蓝牙已启用
- 确保手机与 Ubuntu 设备在同一网络环境中

## 启动节点

### 方法 A：使用启动文件（推荐）

在单个终端中启动所有蓝牙相关节点：

```bash
ros2 launch PipeBot_bringup bluetooth_control.launch.py
```

**预期输出：**
```

[INFO] [bluetooth_receiver_node]: Waiting for mobile app connection...

[INFO] [command_handler_node]: Command handler node started
[INFO] [command_handler_node]: Mode: manual, Timeout: 0.5s
[INFO] [bluetooth_sender_node]: Bluetooth sender node started
```

需要打开 7 个终端：

#### 终端 1：蓝牙接收节点

```bash
ros2 run PipeBot_driver bluetooth_receiver_node

```

```
[INFO] [bluetooth_receiver_node]: Waiting for mobile app connection...
```

```bash

```

```

```

#### 终端 3：命令处理节点

```bash
ros2 run PipeBot_driver command_handler_node

```

## 手机端操作

### 步骤 1：启动蓝牙接收节点

启动节点后，应该看到：

```
[INFO] [bluetooth_receiver_node]: 蓝牙接收节点已启动: PipeBot Control
[INFO] [bluetooth_receiver_node]: 等待手机连接...
[INFO] [bluetooth_receiver_node]: BLE 外设已设置
[INFO] [bluetooth_receiver_node]: 服务 UUID: 12345678-1234-5678-1234-56789ABCDEF0
[INFO] [bluetooth_receiver_node]: 接收特征值 UUID: 12345678-1234-5678-1234-56789ABCDEF1
[INFO] [bluetooth_receiver_node]: 发送特征值 UUID: 12345678-1234-5678-1234-56789ABCDEF2
[INFO] [bluetooth_receiver_node]: BLE 广播已启动
```

### 步骤 2：手机连接

1. 在手机上打开 **nRF Connect** 应用
2. 点击 **"Scan"** 按钮
3. 查找设备名称 **"PipeBot Control"**
4. 点击连接

**预期结果：**
- 设备状态变为 **"Connected"**
- 显示 GATT 服务列表

**Ubuntu 终端输出：**
```
[INFO] [bluetooth_receiver_node]: BLE 广播已启动
```

### 步骤 3：查看服务和特征值

连接后，nRF Connect 会显示：
- **服务 UUID:** 12345678-1234-5678-1234-56789ABCDEF0
- **接收特征值 UUID:** 12345678-1234-5678-1234-56789ABCDEF1 (write)
- **发送特征值 UUID:** 12345678-1234-5678-1234-56789ABCDEF2 (read/notify)

### 步骤 4：验证数据接收

通过 nRF Connect 向接收特征值写入测试数据：

1. 点击接收特征值
2. 点击 "Write"
3. 输入十六进制数据（如 `20 64`）
4. 点击发送

**预期输出（Ubuntu 终端）：**
```
[INFO] [bluetooth_receiver_node]: 接收到命令: 20 64
```

## 发送测试命令

### 方法 A：通过 ROS2 话题发送命令（推荐用于验证）

在新终端中执行以下命令。每个命令对应一个蓝牙控制指令。

#### 前进命令

```bash
ros2 topic pub --once bluetooth/command std_msgs/msg/UInt8MultiArray "data: [32, 100]"
```

**参数说明：**
- `32` (0x20)：前进命令类型
- `100`：速度值（0-255）

**预期输出：**

| 位置 | 输出 |
|------|------|
| **终端 3（命令处理）** | `[INFO] [command_handler_node]: Forward: speed=100` |
| **终端 6（命令监听）** | `data: [32, 100]` |
| **终端 7（状态监听）** | `"velocity": {"linear": 0.196, "angular": 0.0}` |

#### 后退命令

```bash
ros2 topic pub --once bluetooth/command std_msgs/msg/UInt8MultiArray "data: [33, 100]"
```

**参数说明：**
- `33` (0x21)：后退命令类型
- `100`：速度值

**预期输出（终端 3）：**
```
[INFO] [command_handler_node]: Backward: speed=100
```

#### 左转命令

```bash
ros2 topic pub --once bluetooth/command std_msgs/msg/UInt8MultiArray "data: [36]"
```

**参数说明：**
- `36` (0x24)：左转命令类型

**预期输出（终端 3）：**
```
[INFO] [command_handler_node]: Turn left
```

#### 右转命令

```bash
ros2 topic pub --once bluetooth/command std_msgs/msg/UInt8MultiArray "data: [37]"
```

**参数说明：**
- `37` (0x25)：右转命令类型

**预期输出（终端 3）：**
```
[INFO] [command_handler_node]: Turn right
```

#### 制动命令

```bash
ros2 topic pub --once bluetooth/command std_msgs/msg/UInt8MultiArray "data: [34]"
```

**参数说明：**
- `34` (0x22)：制动命令类型

**预期输出（终端 3）：**
```
[INFO] [command_handler_node]: Brake
```

#### 停止命令

```bash
ros2 topic pub --once bluetooth/command std_msgs/msg/UInt8MultiArray "data: [35]"
```

**参数说明：**
- `35` (0x23)：停止命令类型

**预期输出（终端 3）：**
```
[WARN] [command_handler_node]: EMERGENCY STOP
```

#### 自动模式

```bash
ros2 topic pub --once bluetooth/command std_msgs/msg/UInt8MultiArray "data: [16]"
```

**参数说明：**
- `16` (0x10)：自动模式命令类型

**预期输出（终端 3）：**
```
[INFO] [command_handler_node]: Switched to AUTO mode
```

#### 手动模式

```bash
ros2 topic pub --once bluetooth/command std_msgs/msg/UInt8MultiArray "data: [17]"
```

**参数说明：**
- `17` (0x11)：手动模式命令类型

**预期输出（终端 3）：**
```
[INFO] [command_handler_node]: Switched to MANUAL mode
```

### 方法 B：通过手机 nRF Connect 发送命令

> **当前限制：** 由于 `bluetooth_receiver_node.py` 中特征值未完全定义，手机端无法直接通过 nRF Connect 写入数据。需要先完成特征值实现。

## 验证数据流向

### 验证 1：连接状态流向

**目标：** 验证手机连接状态能正确传递到 ROS2 系统

```bash
# 手机连接后，执行
ros2 topic echo bluetooth/status
```

**预期输出：**
```
data: waiting
---
data: connected
```

**手机断开连接后：**
```
data: disconnected
```

### 验证 2：命令处理流向

**目标：** 验证命令能正确从蓝牙接收到速度控制

```bash
# 发送前进命令
ros2 topic pub --once bluetooth/command std_msgs/msg/UInt8MultiArray "data: [32, 100]"

# 在另一个终端查看状态
ros2 topic echo robot/state | grep -A 2 velocity
```

**预期输出：**
```json
"velocity": {
  "linear": 0.196,
  "angular": 0.0
}
```

**数据流向：**
```
手机发送命令 [32, 100]
  ↓
bluetooth/command 话题
  ↓
command_handler_node 接收
  ↓
发布 cmd_vel (linear=0.196, angular=0.0)
  ↓
state_manager_node 订阅 cmd_vel
  ↓
robot/state 包含速度信息
```

### 验证 3：看门狗超时机制

**目标：** 验证命令超时后自动制动功能

```bash
# 发送前进命令
ros2 topic pub --once bluetooth/command std_msgs/msg/UInt8MultiArray "data: [32, 100]"

# 等待 0.6 秒（超过 0.5s 超时时间）
sleep 0.6

# 查看命令处理节点日志
```

**预期输出（终端 3）：**
```
[INFO] [command_handler_node]: Forward: speed=100
[WARN] [command_handler_node]: Command timeout (0.60s), auto brake
```

**说明：** 这个机制确保在手机连接丢失或命令中断时，机器人能自动停止，提高安全性。

### 验证 4：状态聚合

**目标：** 验证所有传感器数据能正确聚合到 robot/state

```bash
ros2 topic echo robot/state
```

**预期输出（完整的 JSON 状态）：**

## 故障排查

### 问题 1：手机扫描不到设备

**症状：** nRF Connect 扫描时看不到 "PipeBot Control" 设备

**可能原因：**
1. 蓝牙硬件未启用
2. bluetooth_receiver_node 未运行
3. 蓝牙适配器故障

**解决方案：**
```bash
# 检查蓝牙硬件
hciconfig

# 启用蓝牙
sudo hciconfig hci0 up

# 检查节点日志
ros2 launch PipeBot_bringup bluetooth_control.launch.py
```

### 问题 2：连接后立即断开

**症状：** 手机能连接但立即断开

**可能原因：**
- 特征值未定义（当前已知限制）
- BLE 服务器配置不完整

**解决方案：**
- 这是当前实现的已知限制，需要完成特征值定义

### 问题 3：收不到命令

**症状：** 发送命令后，bluetooth/command 话题没有数据

**可能原因：**
1. 特征值未定义（当前已知限制）
2. 命令处理节点未运行
3. 话题订阅配置错误

**解决方案：**
```bash
# 检查所有节点是否运行
ros2 node list

# 检查话题是否存在
ros2 topic list

# 通过 ROS2 话题模拟发送命令
ros2 topic pub --once bluetooth/command std_msgs/msg/UInt8MultiArray "data: [32, 100]"
```

### 问题 4：看不到状态反馈

**症状：** 手机无法接收机器人状态信息

**可能原因：**
- `bluetooth_sender_node.py` 中 `send_feedback()` 未完全实现（当前已知限制）
- 特征值未定义

**解决方案：**
- 这是当前实现的已知限制，需要完成反馈发送功能

### 问题 5：节点启动失败

**症状：** 启动节点时出现错误

**常见错误和解决方案：**

```bash
# 错误：ModuleNotFoundError: No module named 'bless'
# 解决：安装依赖
pip install bless pybluez

# 错误：Package not found
# 解决：重新构建
colcon build --packages-select PipeBot_driver PipeBot_bringup
source install/setup.bash

# 错误：Permission denied
# 解决：使用 sudo
sudo ros2 run PipeBot_driver bluetooth_receiver_node
```

## 已知限制

### 1. 蓝牙权限问题

**症状：** 无法打开蓝牙设备

**解决方案：**
```bash
# 添加用户到 bluetooth 组
sudo usermod -a -G bluetooth $USER

# 重新登录或运行
newgrp bluetooth
```

### 2. DBus 连接失败

**症状：** 蓝牙服务无法启动

**解决方案：**
```bash
# 重启蓝牙服务
sudo systemctl restart bluetooth

# 检查蓝牙服务状态
sudo systemctl status bluetooth
```

### 3. 特征值未显示

**症状：** nRF Connect 中看不到特征值

**解决方案：**
- 检查 UUID 格式是否正确
- 确保服务已正确添加
- 查看节点日志输出
