## 2. 核心概念解释

### 2.1 什么是 BLE (蓝牙低功耗)？

- **BLE** = Bluetooth Low Energy，低功耗蓝牙
- 相比经典蓝牙，功耗更低，适合移动设备
- 通信距离：10-100 米
- 数据速率：1 Mbps

### 2.2 BLE 的三层结构

```
┌─────────────────────────────────────────┐
│  应用层 (Application)                   │
│  - 你的业务逻辑（命令处理、状态管理）   │
└─────────────────────────────────────────┘
                    ↕
┌─────────────────────────────────────────┐
│  GATT 层 (Generic Attribute Profile)    │
│  - 定义数据结构                         │
│  - Service（服务）                      │
│  - Characteristic（特征值）             │
│  - Descriptor（描述符）                 │
└─────────────────────────────────────────┘
                    ↕
┌─────────────────────────────────────────┐
│  GAP 层 (Generic Access Profile)        │
│  - 广播（让手机发现设备）               │
│  - 连接管理                             │
└─────────────────────────────────────────┘
```

### 2.3 GATT 中的关键概念

#### Service（服务）
- 一组相关功能的集合
- 用 UUID 标识
- 例如：`12345678-1234-5678-1234-56789ABCDEF0`

#### Characteristic（特征值）
- Service 内的具体数据
- 每个 Characteristic 有：
  - **UUID**：唯一标识符
  - **Value**：实际数据
  - **Properties**：属性（读、写、通知等）

#### 在 PipeBot 中的应用

```
Service: 12345678-1234-5678-1234-56789ABCDEF0
├── Characteristic 1 (接收特征值)
│   ├── UUID: 12345678-1234-5678-1234-56789ABCDEF1
│   ├── Properties: write, write-without-response
│   └── 用途：手机写入命令给树莓派
│
└── Characteristic 2 (发送特征值)
    ├── UUID: 12345678-1234-5678-1234-56789ABCDEF2
    ├── Properties: read, notify
    └── 用途：树莓派发送状态给手机
```

## 4. 关键库和 API

### 4.1 bluezero 库

**作用**：Python 的 BLE 库，简化 BLE 开发

**核心类**：

```python
from bluezero import peripheral

# 创建 BLE 外设
p = peripheral.Peripheral(
    adapter_address='88:A2:9E:4B:BC:E6',  # 蓝牙适配器地址
    local_name='PipeBot Control'            # 设备名称
)

# 添加服务
p.add_service(
    srv_id=1,                               # 服务 ID
    uuid='12345678-1234-5678-1234-56789ABCDEF0',  # 服务 UUID
    primary=True                            # 是否为主服务
)

# 添加特征值
p.add_characteristic(
    srv_id=1,                               # 所属服务 ID
    chr_id=2,                               # 特征值 ID
    uuid='12345678-1234-5678-1234-56789ABCDEF1',  # 特征值 UUID
    value=b'',                              # 初始值
    notifying=False,                        # 是否启用通知
    flags=['write', 'write-without-response'],  # 属性
    write_callback=callback_function        # 写入回调
)

# 启动广播
p.publish()
```

## 5. 数据流示例

### 5.1 手机发送命令给树莓派

```
手机 (nRF Connect)
    ↓ 写入数据到接收特征值
    ↓ [0x01, 0x02, 0x03]
    ↓
树莓派 (bluetooth_receiver_node)
    ↓ _on_write_request() 被触发
    ↓ 创建 UInt8MultiArray 消息
    ↓ 发布到 /bluetooth/command 话题
    ↓
其他节点 (command_handler_node)
    ↓ 订阅 /bluetooth/command
    ↓ 处理命令
    ↓ 执行相应动作
```

### 5.2 树莓派发送状态给手机

```
其他节点 (state_manager_node)
    ↓ 发布状态到 /bluetooth/feedback 话题
    ↓ [0xAA, 0x01, 0x0C, ...]
    ↓

    ↓ feedback_callback() 被触发
    ↓ 更新发送特征值的值
```

## 7. 常见问题解答

### Q1: 为什么需要 GLib 主循环？
**A**: bluezero 使用 D-Bus 与系统蓝牙服务通信。D-Bus 是异步的，需要主循环来处理事件。

### Q2: UUID 是什么？
**A**: 通用唯一标识符，用来唯一标识 BLE 服务和特征值。格式：`xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`

### Q3: 为什么有两个特征值？
**A**: 
- 接收特征值：手机 → 树莓派（手机写入，树莓派读取）
- 发送特征值：树莓派 → 手机（树莓派写入，手机读取）

### Q4: 什么是 notify（通知）？
**A**: BLE 的一种通信方式。树莓派主动推送数据给手机，而不是等待手机查询。

### Q5: 为什么要用线程运行 BLE 服务器？
**A**: BLE 主循环是阻塞的，需要在单独的线程中运行，以免阻塞 ROS2 的主循环。
