# PipeBot 视觉流媒体系统

## 📋 概述

通过以太网将USB相机采集的1080p高清图像实时传输至中控平台（上位机），用于前端网页显示和缺陷检测推理。

**系统架构：**
```
USB相机 (1920×1080@30fps)
    ↓
image_capture_node (ROS2)
    ↓
web_video_server (HTTP/MJPEG)
    ↓
中控平台 (上位机)
    ├→ 前端网页显示
    ├→ 缺陷检测推理
    └→ 云端上传
```

## 🎯 性能指标

| 指标 | 值 |
|------|-----|
| **分辨率** | 1920×1080 |
| **采集帧率** | 30fps |
| **传输帧率** | 5fps |
| **编码格式** | MJPEG |
| **传输质量** | 80 |
| **网络带宽** | ~0.54 Mbps |
| **树莓派CPU占用** | 11-15% |
| **端到端延迟** | 200-500ms |
| **服务端口** | 8080 |

## 📦 依赖安装

### 1. 安装web_video_server

```bash
sudo apt update
sudo apt install ros-humble-web-video-server
```

### 2. 构建PipeBot_vision包

```bash
cd ~/PipeBot_ws
colcon build --packages-select PipeBot_vision
source install/setup.bash
```

## 🚀 启动方式

### 方式一：使用launch文件（推荐）

```bash
# 源环境
source ~/PipeBot_ws/install/setup.bash

# 启动视觉流媒体系统
ros2 launch PipeBot_vision vision_streaming.launch.py
```

### 方式二：手动启动各节点

```bash
# 终端1：启动image_capture_node
ros2 run PipeBot_vision image_capture_node

# 终端2：启动web_video_server
ros2 run web_video_server web_video_server
```

## ⚙️ 参数配置

### image_capture_node参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `camera_index` | 0 | USB相机索引 |
| `frame_width` | 1920 | 采集帧宽度 |
| `frame_height` | 1080 | 采集帧高度 |
| `fps` | 30 | 相机采集帧率 |
| `publish_rate` | 30 | ROS话题发布频率 |
| `jpeg_quality` | 85 | JPEG压缩质量 |

### web_video_server参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `port` | 8080 | HTTP服务端口 |
| `quality` | 80 | 网络传输JPEG质量 |
| `default_stream_type` | mjpeg | 流媒体类型 |
| `max_streams` | 5 | 最大并发连接数 |
| `bitrate_kbps` | 600 | 目标码率（Kbps） |

### 自定义参数启动

```bash
ros2 launch PipeBot_vision vision_streaming.launch.py \
    camera_index:=0 \
    frame_width:=1920 \
    frame_height:=1080 \
    fps:=30 \
    jpeg_quality:=85 \
    web_port:=8080 \
    web_quality:=80
```

## 📊 ROS话题

### 发布话题

| 话题 | 消息类型 | 频率 | 说明 |
|------|---------|------|------|
| `/camera/image_raw` | `sensor_msgs/Image` | 30fps | 原始1080p图像 |

### 订阅话题

web_video_server订阅`/camera/image_raw`话题，并通过HTTP/MJPEG提供流媒体服务。

## 🔍 故障排查

### 问题1：相机无法打开

**症状：** 日志显示 `Failed to open camera 0`

**解决方案：**
```bash
# 检查相机设备
ls -la /dev/video*

# 如果有多个相机，尝试其他索引
ros2 run PipeBot_vision image_capture_node --ros-args -p camera_index:=1
```

### 问题2：网页无法连接

**症状：** 浏览器显示 `连接失败`

**解决方案：**
```bash
# 检查web_video_server是否运行
ros2 node list | grep web_video_server

# 检查端口是否被占用
sudo netstat -tlnp | grep 8080

# 检查防火墙
sudo ufw allow 8080
```

### 问题3：帧率过低

**症状：** 网页显示帧率 < 5fps

**解决方案：**
1. 检查树莓派CPU占用：`top`
2. 降低JPEG质量：`web_quality:=70`
3. 降低传输帧率：修改launch文件中的`bitrate_kbps`

### 问题4：网络延迟高

**症状：** 延迟 > 1000ms

**解决方案：**
1. 使用有线以太网连接（避免WiFi）
2. 检查网络带宽：`iftop`
3. 降低JPEG质量或帧率

## 🔧 高级配置

### 调整网络带宽

如果网络带宽有限，可以降低传输质量：

```bash
ros2 launch PipeBot_vision vision_streaming.launch.py \
    web_quality:=60 \
    web_port:=8080
```

**带宽估算：**
- 质量60：~0.35 Mbps
- 质量70：~0.45 Mbps
- 质量80：~0.54 Mbps
- 质量90：~0.70 Mbps

### 多客户端连接

web_video_server支持多个客户端同时连接：

```bash
# 修改launch文件中的max_streams参数
ros2 launch PipeBot_vision vision_streaming.launch.py \
    web_port:=8080
```

## 🐛 已知问题

1. **MJPEG延迟**：MJPEG格式每帧独立编码，延迟通常在200-500ms
2. **带宽消耗**：相比H.264，MJPEG带宽消耗较大
3. **浏览器兼容性**：某些旧版浏览器可能不支持MJPEG

3. 网络连接是否正常
4. 防火墙是否允许8080端口
